## 内容
- Origin システムの実装開始: 入力・管理インターフェースの整備
  これが自己のデータとなり実行(Act)への意思決定(Decide)を最小化する
- GitHub リポジトリ「Life」への構造的集約とObsidian/Cursor 連携強化
  こうすることで人生そのものを管理することが可能となる

### 1. Zettelkasten のデジタル実装 (Obsidian × Cursor)
- "リテラチャーノートの自動蓄積":
    - "Web記事や動画情報を 'save to notion' や API 経由で取得し、
    - Markdown として '/data/raw/literature/' へ自動保存する仕組み検討
    - こうすることで、探す作業を省略する
    - ノート同士のリンクを網目状に張り巡らせ、脳のネットワーク構造模倣
    - 整理しかつ網目状にすることで、新しいアイデアが生まれやすくなる
- "パーマネントノートの「自己完結性」徹底"
    - コピペを禁止し、「自分の言葉」で1トピック1ファイル最小単位に要約
    - このことで、言語化をする具体性が実行される
    - CursorのAI補完を活用、TILやメモの「点」を繋ぎ、パーマネントへ昇華
    - この具体化からの抽象化の運動が重要になってくる

### 2. GitHub による「人生のOS」運用の具体化
- "WIP (仕掛品) 制限の実装":
    - GitHub Projects の Kanban を使用し、In Progress 5個、全体30個制限
    - 散漫を防ぐため、30を超えるIssueはバックログへ運用徹底
- "コンテキストスイッチの最小化":
    - 投資・学習・事業・趣味のすべてを一つの「Life」リポジトリにIssue集約
    - 「人生の中に事業が内包」の思想に基づき、意思決定を削減化

### 3.データ駆動循環(Obsidian -> Orient) の強化
- "ETL パイプラインの前工程":
    - SQLite / DuckDB への TIL データの自動ロードを検討
    - Claude Code を使用し、'daily_log' から重要なキーワード抽出
    - Zettelkasten の「インデックス(タグ)」自動付与するプロンプト試作
