## 内容
- データ駆動循環の実装: Markdownから構造的データへの変換(Extract)
    - TIL から Issue への自動起票の具体化
    - TIL (非構造化データ) からメタデータ(YAML Frontmatter)を抽出、
    　SQLite/DuckDBへ流し込むスキーマ設計
    - 学びのポイント: Daily Log をCategory, Tag, Time_Spentを
    　YAMLで記述する習慣により、分析(Polars)に役立つ。
- Zettelkastenの深化: インデックスノート(MOC) の作成
    - 散らばったリテラチャーノートを繋ぐMOC(Map of Content)を作成
    - Life管理、技術スタック、投資戦略などのハブとなるノートを配置し、
    　ネットワークの結節点を強化
    - Cursor AI, Claude Code がコンテキストを読み取るとき、
    　MOCを参照することで関連知識の引き出し向上する
- GitHubを用いた「戦略型習慣」の自動トリガー
    - GitHub Actionsの定期実行(cron)を用いて、前日のIssueの残り数を
    　を通知する仕組みを検討する。
    - WIP制限(5個)を超えている場合、新規Issue作成に対する
    　心理的ブレーキシステムを実装。
