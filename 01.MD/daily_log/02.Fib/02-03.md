## 内容
- URL を貼って見た
- Note にOrigin TIL を投稿した
- Origin システムのデータ駆動循環の設計
    - 1.Zettelkasten による「点」から「線」への知識構造化
        - 3 つのノートの役割: メモを「知的資産」にするには階層分離が重要
            - リテラチャーノート(文献メモ): 外部情報のストック
            - パーマネントノート(永久ノート): 自分の言葉で書きかえた
            　「自己完結する知識の最小単位」
            - 一環(インデックス): 知識同士をリンク、ネットワーク構造構築
        - 学びの原則: 必ず「自分の言葉」で要約する
        　記憶の定着と意思決定のスピードが向上する
    - 2.GitHubを用いた「人生のOS」構築と自動化
        - コンテキストスイッチの排除: 仕事も個人プロジェクトも投資も、
        　すべて一つの「Lifeリポジトリ」に集約することでコスト最小化
        - issue を活用したタスク管理
            - 仕掛け中のタスク(In Progress) は5個、全体のストックは30個
            　""WIP制限(仕掛け制限)を設けることで、注意力散漫を防ぐ
            - Claude Code を活用し、Issueの自動起票や情報の要約を行い
            　入力負荷を軽減する
    - 3.「Origin」アーキテクチャへの統合
        - 観察(Observe) -> TILの役割: 1回5分のTILはシステム最小入力単位
        　「リテラチャーノート」「エージェントメモリー」となる。
        - 方向(Orient) -> ETLと分析: SQLiteやDuckDBに蓄積されたTILの記録
        　をPython/Polarsで処理し、可視化(Grafana/Streamilt)することで、
        　現状の客観化を目指す。
        - 意思決定(Decide): 構造化されたデータに基づき、「戦略型習慣」
        　として月次レポートやMLによる予測からAction(実行)を導き出す
